Review of elements used for the widget

First off, let’s look at the basic markup already in place.

The widget is placed inside a <form> component.

<form action="#">
  <label for="mammals">Favourite mammal</label>
  <input id="mammals" type="text">
  <button type="button"></button>
  <ul id="mammals-list">
    <li>Aardvark</li>
    <li>African Elephant</li>
    <!-- ... -->
  </ul>
</form>

There is a <label> to provide an accessible name for the <input>.

<form action="#">
  <label for="mammals">Favourite mammal</label>
  <input id="mammals" type="text">
  <button type="button"></button>
  <ul id="mammals-list">
    <li>Aardvark</li>
    <li>African Elephant</li>
    <!-- ... -->
  </ul>
</form>

The <input> is programatically associated with the <label> via matching for and id values.

<form action="#">
  <label for="mammals">Favourite mammal</label>
  <input id="mammals" type="text">
  <button type="button"></button>
  <ul id="mammals-list">
    <li>Aardvark</li>
    <li>African Elephant</li>
    <!-- ... -->
  </ul>
</form>

The <button> is used to trigger the dropdown.

<form action="#">
  <label for="mammals">Favourite mammal</label>
  <input id="mammals" type="text">
  <button type="button"></button>
  <ul id="mammals-list">
    <li>Aardvark</li>
    <li>African Elephant</li>
    <!-- ... -->
  </ul>
</form>

The <ul> displays possible results when triggered or when results are available.

<form action="#">
  <label for="mammals">Favourite mammal</label>
  <input id="mammals" type="text">
  <button type="button"></button>
  <ul id="mammals-list">
    <li>Aardvark</li>
    <li>African Elephant</li>
    <!-- ... -->
  </ul>
</form>

Adding accessibility

Step 1:

Add role="combobox" to <input>.

This will change the input element's role from textbox to combobox.

<input
  id="mammals" 
  type="text"
  role="combobox"
>

Step 2:

Add aria-autocomplete="list" to <input>.

When a user is providing input, an element containing a list of values that could complete the provided input may be displayed.

<input
  id="mammals" 
  type="text"
  role="combobox"
  aria-autocomplete="list"
>

Step 3:

Add aria-expanded="false" to <input>.

The true/false value is toggled based on expanded/collapsed state.

<input
  id="mammals" 
  type="text"
  role="combobox"
  aria-autocomplete="list"
  aria-expanded="false"
>

Step 4:

Add aria-controls="..." to <input>.

This connects the input field with the collection of options.

<input
  id="mammals" 
  type="text"
  role="combobox"
  aria-autocomplete="list"
  aria-expanded="false"
  aria-controls="mammals-list"
>

Step 5:

Add tabindex="-1" to the trigger <button>.

This takes the button out of the tab order. Keyboard users can use arrow keys. Mouse users can still click the button.

<button 
  type="button" 
  tabindex="-1"
>
  <!-- ... -->
</button>

Step 6:

Add aria-label="Mammals" to <button>.

The gives the button an accessible name.

<button 
  type="button"
  tabindex="-1"
  aria-label="Mammals"
>
  <!-- ... -->
</button>

Step 7:

Add aria-expanded="false" to <button>.

The true/false value is toggled based on expanded/collapsed state.

<button 
  type="button"
  tabindex="-1"
  aria-label="Mammals"
  aria-expanded="false"
>
<!-- ... -->
</button>

Step 8:

Add aria-controls="mammals-list" to <button>.

This connects the button element to the collection of options.

<button 
  type="button"
  tabindex="-1"
  aria-label="Mammals"
  aria-expanded="false"
  aria-controls="mammals-list"
>
<!-- ... -->
</button>

Step 9:

Add id="mammals-list" to the <ul>.

The ID value is referenced by aria-controls on the input and button.

<ul id="mammals-list">
  <li>Aardvark</li>
  <!-- ... -->
</ul>

Step 10:

Add role="listbox" to the <ul>.

This will change the role of the ul from list to listbox

<ul
  id="mammals-list"
  role="listbox"
>
  <li>Aardvark</li>
  <!-- ... -->
</ul>

Step 11:

Add aria-label="Mammals" to the <ul>.

This will give the listbox an accessible name.

<ul
  id="mammals-list"
  role="listbox"
  aria-label="Mammals"
>
  <li>Aardvark</li>
  <!-- ... -->
</ul>

Step 12:

Add role="option" to each of the <li> items.

This changes the role of the li from listitem to option.

<ul id="mammals-list" role="listbox" aria-label="Mammals">
  <li role="option">Aardvark</li>
  <li role="option">African Elephant</li>
  <!-- ... -->
</ul>

JavaScript dynamically manages:

  • aria-selected="true/false" on the list items.
  • aria-activedescendant="ID" on the input
<input aria-activedescendant="op-5" id="mammals" class="input" type="text" autocomplete="off" role="comobox" aria-autocomplete="list" aria-expanded="true" aria-controls="mammals-list">
<!-- ... -->
<ul>
<!-- ... -->
  <li aria-selected="true" id="op-5" ...>Arctic Fox</li>
</ul>
